## 幻灯片 2：项目概况与系统架构
**1. 项目背景**
*   公共安全监控智能化需求日益增长，传统人工监控存在滞后性与易疲劳问题。
*   旨在开发一套能够利用现有监控摄像流，实时识别并预警暴力冲突行为的自动化系统。

**2. 核心技术栈**
*   **前端交互**：Streamlit (构建实时可视化仪表盘与报警反馈机制)
*   **特征提取**：YOLOv11-Pose (高性能人体骨架关键点提取)
*   **行为分类**：MindSpore (国产深度学习框架，构建时序分类模型)

**3. 系统流程 (Pipeline)**
*   **数据采集**：支持 RTSP 实时视频流/本地视频文件输入。
*   **预处理**：利用 YOLOv11n-pose 实时提取画面中多目标人体的 17 个关键骨架点。
*   **推理判决**：构建时序动作分类网络，对骨架序列进行二分类判定 (暴力/非暴力)。

## 幻灯片 3：技术演进路线 (Iterative Development)
本项目非一蹴而就，通过三个阶段的迭代实验，逐步确定了最终的技术方案：

**阶段一：基于 CNN 的单帧图像分类（初期探索）**
*   **尝试方案**：将视频帧作为独立图像输入卷积神经网络。
*   **局限性分析**：
    *   **时序信息缺失**：无法区分静态姿势（如拥抱）与动态攻击动作。
    *   **背景噪声干扰**：复杂背景严重影响特征提取的准确性。
*   **结论**：暴力检测本质是动作识别，必须引入时序维度与骨架特征。

**阶段二：Keras + OpenPose 方案（性能瓶颈）**
*   **尝试方案**：引入骨架提取算法，利用 LSTM/GRU 处理时序数据。
*   **存在问题**：
    *   **实时性差**：OpenPose 推理延迟高，无法满足实时监控需求。
    *   **IO 瓶颈**：原始视频训练导致数据加载效率极低。
*   **优化措施**：开发数据预处理脚本 (`utills/data_preprocessing.py`)，将视频流离线转换为 Numpy 格式特征文件 (`.npz`)，显著提升训练效率。

**阶段三：YOLOv11 + MindSpore 方案（最终定型）**
*   **升级方案**：采用轻量级 YOLOv11n-pose 替代 OpenPose，推理速度提升数倍。
*   **框架迁移**：基于国产化适配需求，将分类模型迁移至 MindSpore 框架 (`version3.mindir`)，实现高效推理。

## 幻灯片 4：关键技术挑战与工程化解决方案 (1/2)
**挑战一：检测结果的稳定性（“闪烁”问题）**
*   **问题描述**：单帧推理受视角或遮挡影响，导致分类结果在“暴力/非暴力”间高频跳变。
*   **解决方案**：**滑动窗口平滑算法 (Smoothing Window)**
    *   引入队列机制，缓存历史 N 帧（当前设置为 5-10 帧）的预测概率。
    *   采用加权平均策略输出最终判定结果，设定高置信度阈值（Threshold），有效过滤偶发性误判，提升系统鲁棒性。

## 幻灯片 5：关键技术挑战与工程化解决方案 (2/2)
**挑战二：复杂场景下的误报控制 (False Positives)**
*   **问题描述**：高相似度动作（如拥抱、击掌、舞蹈）常被误识别为暴力行为。
*   **解决方案**：**难例挖掘 (Hard Negative Mining)**
    *   构建针对性的非暴力数据集，引入大量具有复杂肢体接触的非暴力样本（“强干扰样本”）。
    *   重训练模型以强化特征边界，显著提升了模型对亲密行为与攻击行为的辨别能力。

**挑战三：异构框架资源调度**
*   **问题描述**：PyTorch (YOLO) 与 MindSpore 在同一运行时环境中并存导致 GPU 显存竞争与上下文冲突。
*   **解决方案**：**异构计算资源分配**
    *   实施计算分流策略：YOLO 负责视觉计算（可分配至 GPU），MindSpore 负责逻辑推理（配置为 `device_target='CPU'`）。
    *   通过解耦计算负载，规避了框架间的资源死锁，确保了系统运行的流畅性。
